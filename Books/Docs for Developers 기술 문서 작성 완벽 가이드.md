
## 2. 문서화 계획하기

- 사용자에게 가장 중요한 유즈 케이스를 파악하는게 우선
- 사용자의 니즈를 해결하는 콘텐츠 유형으로 문서화 계획을 짜야한다. 

### 콘텐츠 유형

#### 코드 주석
- 과거에 내린 결정의 맥락
- 왜 이런 코드 스타일을 결정했는지, 어떤 요구사항이 있었는지 등
- 중요하게 여겨야할 것은, 다음에 이 코드를 보는 사람은 해당 코드에 대한 아무런 정보가 없다는 사실. 

#### README
- README는 최대한 간결하고 유익하며, 정확하고, 최신 정보를 담고 있어야 한다.
- 포함되어야하는 단락 예시
	- 문제 해결 단계
	- 코드 유지 관리자
	- 라이선스 정보
	- 변경 로그
	- 기본 사용 예시
	- 심화 자료와 문서 링크

#### 시작하기 문서



- 코드 주석
	- 과거에 내린 결정의 맥락
	- 왜 이런 코드 스타일을 결정했는지, 어떤 요구사항이 있었는지 등 유지보수에 필요한 내용들

- README
	- 포함되어야하는 단락 예시
		- 문제 해결 단계
		- 코드 유지 관리자
		- 라이선스 정보
		- 변경 로그(change log)
		- 기본 사용 예시
		- 심화 자료와 문서 링크
	- README는 최대한 간결하고 유익하며 정확하고 최신 정보를 담고 있어야 한다.
- 시작하기 문서
- 개념 문서
- 절차 문서
	- 튜토리얼
	- How-to 가이드
- 참조 문서
	- API 레퍼런스
	- 용어집
	- 문제 해결 문서
