## 빌더 패턴의 두가지 종류

빌더 패턴을 사용하는데 있어서 해당 내용이 가장 골치아픈 내용이라고 생각한다. 

빌더 패턴에 대한 개념은 크게 이펙티브 자바에서의 개념과 GoF의 개념, 두가지로 나뉜다. 

- **이펙티브 자바의 빌더 패턴** : 생성시 지정해야 할 인자가 많을때 사용. 객체의 일관성 불변성이 목적.
- **GoF의 빌더 패턴** : 객체의 생성 단계 순서를 결정해두고 각 단계를 다양하게 구현하고 싶을때 사용.

사실 이 두 책이 나온 시점이 꽤 간극이 크고, 목적성도 다르다보니 개념적으로도 차이가 날 수밖에 없다고 생각하지만, 관련 개념에 대해 커뮤니케이션을 진행할때 분명히 문제가 발생할 수 있는 지점이다보니 짚고 넘어가는게 좋다고 생각한다. 


### 심플 빌더 패턴

- **디렉터 불필요**: 심플 빌더 패턴에서는 디렉터 객체가 없습니다. 클라이언트가 직접 빌더를 사용하여 필요한 객체의 생성을 제어한다.
- **구현 간소화**: 클라이언트 코드는 빌더 인터페이스를 직접 사용하여 생성 과정을 특정한다. 이는 빌더 자체가 생성 로직의 모든 세부 사항을 캡슐화하고 있음을 의미한다.
- **사용 용이성**: 빌더만으로도 충분히 간단한 상황이나 더 직관적이고 유연한 방식이 필요할 때 유리하다.

### 디렉터 빌더 패턴

- **디렉터 필요**: 디렉터 객체가 존재하며, 이 객체가 빌더를 조정하여 객체의 생성 과정을 지휘한다.
- **복잡한 생성 과정 관리**: 디렉터는 생성 과정의 각 단계를 정의하고, 빌더는 이 단계에 맞춰 구체적인 구성 요소를 생성한다. 따라서, 동일한 생성 과정을 사용하여 다양한 형태의 객체를 생산할 수 있다. 
- **재사용성과 일관성**: 디렉터를 사용함으로써 빌더를 통한 객체 생성 과정을 표준화할 수 있다. 이는 코드의 재사용성과 일관된 객체 생성 로직을 보장한다.

###  차이점

- **제어의 중앙화 여부**: 심플 빌더 패턴은 클라이언트가 모든 생성 로직을 직접 제어하지만, 디렉터 빌더 패턴에서는 디렉터가 이 역할을 중앙집중화하여 수행한다. 이로 인해 디렉터 빌더 패턴은 더 복잡한 시나리오에서 생성 과정의 일관성을 유지하는 데 더 효과적이다.
- **객체 생성의 복잡성**: 디렉터 빌더 패턴은 복잡한 객체 생성 과정을 효과적으로 관리할 수 있으며, 다양한 구성 요소와 단계를 통합하는 큰 규모의 애플리케이션에 적합하다. 반면, 심플 빌더 패턴은 보다 간단하거나 일시적인 객체 생성에 적합할 수 있다.


lombok에서도 @Builder 어노테이션을 통해 빌더 패턴을 지원하는데, 해당 방식은 심플 빌더 패턴을 따르고 있다. 



### 심플 빌더 패턴

이펙티브 자바에서 소개한 빌더 패턴이며, 롬복에서 **@Builder** 어노테이션을 사용할 경우 구현되는 방식이다. 
일반적으로 이야기하는 빌더 패턴이 이 방식이라고 볼 수 있다. 


```java
class Person {
	String name;
	int age;
	
	public static class Builder {
		String name;
		int age;
		
		Builder name(String name) {
			this.name = name;
			return this;
		}
		
		Builder age(int age) {
			this.age = age;
			return this;
		}
	}
	
	private Person(Builder builder) {
		this.name = builder.name;
		this.age = builder.age;
	}
}
```


### 디렉터 빌더 패턴

빌더 패턴의 장점 중에 객체 생성 지연이라는 점을 언급했었다. 
```java
// 1. 빌더 클래스 전용 리스트 생성
List<StudentBuilder> builders = new ArrayList<>();
// 2. 객체를 최종 생성 하지말고 초깃값만 세팅한 빌더만 생성
builders.add(
    new StudentBuilder(2016120091)
    .name("홍길동")
);
builders.add(
    new StudentBuilder(2016120092)
    .name("임꺽정")
    .grade("senior")
);
builders.add(
    new StudentBuilder(2016120093)
    .name("박혁거세")
    .grade("sophomore")
    .phoneNumber("010-5555-5555")
);
// 3. 나중에 빌더 리스트를 순회하여 최종 객체 생성을 주도
for(StudentBuilder b : builders) {
    Student student = b.build();
    System.out.println(student);
}
```

위 방식처럼 속성에 대한 지정은 해주되, 객체를 생성하는 시점만 뒤로 늦추는 방식인데, 디렉터 빌더 패턴에서도 비슷하게 구현되는 느낌이다. 

빌드 용도의 클래스를 만들어두고, 해당 클래스를 상속받아서 객체를 

```java
class Data {
    private String name;
    private int age;

    public Data(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public String getName() {
        return name;
    }

    public int getAge() {
        return age;
    }
}
```

```java
abstract class Builder {
    // 상속한 자식 클래스에서 사용하도록 protected 접근제어자 지정
    protected Data data;

    public Builder(Data data) {
        this.data = data;
    }

    // Data 객체의 데이터들을 원하는 형태의 문자열 포맷을 해주는 메서드들 (머리 - 중간 - 끝 형식)
    public abstract String head();
    public abstract String body();
    public abstract String foot();

}
```

```java
// Data 데이터들을 평범한 문자열로 변환해주는 빌더
class PlainTextBuilder extends Builder {

    public PlainTextBuilder(Data data) {
        super(data);
    }

    @Override
    public String head() {
        return "";
    }

    @Override
    public String body() {
        StringBuilder sb = new StringBuilder();

        sb.append("Name: ");
        sb.append(data.getName());
        sb.append(", Age: ");
        sb.append(data.getAge());

        return sb.toString();
    }

    @Override
    public String foot() {
        return "";
    }
}
```

```java
// Data 데이터들을 JSON 형태의 문자열로 변환해주는 빌더
class JSONBuilder extends Builder {

    public JSONBuilder(Data data) {
        super(data);
    }

    @Override
    public String head() {
        return "{\n";
    }

    @Override
    public String body() {
        StringBuilder sb = new StringBuilder();

        sb.append("\t\"Name\" : ");
        sb.append("\"" + data.getName() + "\",\n");
        sb.append("\t\"Age\" : ");
        sb.append(data.getAge());

        return sb.toString();
    }

    @Override
    public String foot() {
        return "\n}";
    }
}
```

```java
// Data 데이터들을 XML 형태의 문자열로 변환해주는 빌더
class XMLBuilder extends Builder {
    public XMLBuilder(Data data) {
        super(data);
    }

    @Override
    public String head() {
        StringBuilder sb = new StringBuilder();

        sb.append("<?xml version=\"1.0\" encoding=\"UTF-8\" ?>\n");
        sb.append("<DATA>\n");

        return sb.toString();
    }

    @Override
    public String body() {
        StringBuilder sb = new StringBuilder();

        sb.append("\t<NAME>");
        sb.append(data.getName());
        sb.append("<NAME>");
        sb.append("\n\t<AGE>");
        sb.append(data.getAge());
        sb.append("<AGE>");

        return sb.toString();
    }

    @Override
    public String foot() {
        return "\n</DATA>";
    }
}
```

```java
// 각 문자열 포맷 빌드 과정을 템플릿화 시킨 디렉터
class Director {
    private Builder builder;

    public Director(Builder builder) {
        this.builder = builder;
    }

    // 일종의 빌드 템플릿 메서드라 보면 된다
    public String build() {
        StringBuilder sb = new StringBuilder();

		// 빌더 구현체에서 정의한 생성 알고리즘이 실행됨
        sb.append(builder.head());
        sb.append(builder.body());
        sb.append(builder.foot());

        return sb.toString();
    }
}
```

```java
public static void main(String[] args) {
    // 1. 포맷할 자바 데이터 생성
    Data data = new Data("홍길동", 44);

    // 2. 일반 텍스트로 포맷하여 출력하기
    Builder builder1 = new PlainTextBuilder(data);
    Director director1 = new Director(builder1);
    String result1 = director1.build();
    System.out.println(result1);

    // 3. JSON 형식으로 포맷하여 출력하기
    Builder builder2 = new JSONBuilder(data);
    Director director2 = new Director(builder2);
    String result2 = director2.build();
    System.out.println(result2);

    // 4. XML 형식으로 포맷하여 출력하기
    Builder builder3 = new XMLBuilder(data);
    Director director3 = new Director(builder3);
    String result3 = director3.build();
    System.out.println(result3);
}
```
